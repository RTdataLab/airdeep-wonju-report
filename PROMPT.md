# 매달 업데이트 프롬프트 — 원주시설관리공단

데이터팀이 준 이번 달 CSV를 이 폴더에 두고, 아래 박스 안의 글을 복사해서 Codex에 붙여넣으세요.
맨 윗줄의 달과 공휴일만 그 달에 맞게 바꾸면 됩니다.

```
2026년 6월 데이터입니다. 아래대로 리포트를 업데이트해줘.

1. 데이터 교체
   내가 준 CSV를 data 폴더에 아래 이름으로 덮어쓰기 해줘.
   - 3-1.csv : 구역별 일평균 실내온도
   - 3-2.csv : 온습도계 설치 구역 온도
   - 4-1.csv : 구역별 누적 가동시간
   - 5-1.csv : 종합운동장 공간별 가동시간
   - 5-2.csv : 치악체육관 공간별 가동시간
   - 5-3.csv : 국민체육센터 층별 가동시간
   - 5-4.csv : 종합체육관 공간별 가동시간
   - 6-1.csv : 구역별 종합 분석표
   - 6-2.csv : 월간 누적 가동시간 TOP5
   파일명이 다르면 내용을 보고 맞는 번호로 넣고, 컬럼명이 기존과 다르면 app.js가 그 컬럼도 읽도록 보완해줘.
   디자인이나 레이아웃, 파일 구조(섹션-그래프 규칙)는 바꾸지 마.

2. 수치와 코멘트 다시 쓰기
   숫자는 반드시 CSV에서 직접 계산하고 임의로 지어내지 마.
   섹션1의 KPI 4개와 주의 항목 3건, 총평, 각 차트 아래 코멘트, 섹션6의 종합 분석표와 TOP5 표 코멘트 새 데이터 기준으로 다시 써줘.
   이번 달 가장 눈에 띄는 부분(상시 가동 공간, 급증 구역, 고온 구간 등)을 중심으로 정리해줘.

3. 기간과 공휴일
   헤더의 기간을 이번 달(예: 2026.06.01 — 2026.06.30)로 바꿔줘.
   공휴일은 네가 먼저 이 데이터가 속한 연·월의 한국 공휴일을 정리해서
   "이번 달 공휴일은 O/O(공휴일명)입니다. 맞나요?" 라고 나한테 확인받아줘.
   내가 확인해주면 그 날짜만 app.js 맨 위 PUBLIC_HOLIDAYS에 넣어줘.
   (토·일 주말은 자동 처리되니 평일 공휴일만 넣으면 돼. 확실하지 않으면 임의로 넣지 말고 반드시 먼저 물어볼 것)

4. index.html의 app.js?v= 와 style.css?v= 버전을 오늘 날짜로 올려줘.

5. 다 되면 커밋하고 origin main에 push해줘. 그리고 옛 숫자가 남지 않았는지 확인한 다음 라이브 주소를 알려줘.
```
