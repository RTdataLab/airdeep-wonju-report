# 📋 매달 리포트 업데이트 프롬프트 — 원주시설관리공단

## 사용법
1. 데이터팀이 준 이번 달 CSV 파일들을 이 폴더(또는 data 폴더)에 둡니다.
2. 아래 **회색 박스 안 글을 통째로 복사**해서 Codex 대화창에 붙여넣습니다.
3. 맨 윗줄의 **[이번 달]** 과 **공휴일** 부분만 그 달에 맞게 고칩니다.
4. Codex가 데이터 교체 → 수치·코멘트 재작성 → push 까지 진행합니다.

---

## ⬇️ 매달 복붙용 프롬프트 (이 박스 안만 복사)

```
[이번 달] 2026년 6월 데이터입니다. 아래 기준으로 이 리포트를 업데이트해줘.

1) 데이터 교체
   내가 준 CSV들을 data/ 폴더에 아래 규칙 이름으로 덮어쓰기 해줘.
   - 3-1.csv = 구역별 일평균 실내온도
   - 3-2.csv = 온습도계 설치 구역 온도
   - 4-1.csv = 구역별 누적 가동시간
   - 5-1.csv = 종합운동장 공간별 가동시간
   - 5-2.csv = 치악체육관 공간별 가동시간
   - 5-3.csv = 국민체육센터 층별 가동시간
   - 5-4.csv = 종합체육관 공간별 가동시간
   - 6-1.csv = 구역별 종합 분석표
   - 6-2.csv = 월간 누적 가동시간 TOP5
   - 파일명이 다르면 내용을 보고 알맞은 번호로 매칭해서 넣어줘.
   - CSV 컬럼명이 기존과 다르면 app.js가 그 컬럼도 읽도록 보완해줘.
   - ※ 디자인·레이아웃·파일 구조(섹션-그래프 규칙)는 절대 바꾸지 말 것.

2) 수치·코멘트 재작성 (숫자는 반드시 CSV에서 직접 계산, 절대 지어내지 말 것)
   - 섹션1: KPI 4개 값, "주의가 필요한 항목" 3건, 총평
   - 각 차트 아래 코멘트(분석 글)
   - 섹션6의 종합 분석표와 TOP5 표 코멘트
   을 새 데이터 기준으로 다시 써줘.
   이번 달 가장 두드러진 이슈(상시 가동 공간, 급증 구역, 고온 구간 등)를 핵심으로 다뤄줘.

3) 기간·공휴일
   - 헤더 기간(예: 2026.06.01 — 2026.06.30)을 이번 달로 바꿔줘.
   - app.js 맨 위 PUBLIC_HOLIDAYS에 이번 달 공휴일을 넣어줘 (주말은 자동 계산).
     이번 달 공휴일: 6/6 현충일      ← 그 달 공휴일로 수정

4) 캐시 버전
   - index.html의 app.js?v= 와 style.css?v= 값을 오늘 날짜로 올려줘.

5) 마무리
   - 변경사항을 커밋하고 origin main 에 push 해줘.
   - 옛 수치가 남아있지 않은지 점검하고, 라이브 주소를 알려줘.
```
