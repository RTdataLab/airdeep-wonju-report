# 원주시설관리공단 월간 모니터링 리포트

실내 온습도와 냉난방기 가동을 정리한 리포트입니다. HTML/CSS/CSV로 만든 정적 페이지라,
매달 data 폴더의 CSV만 바꾸면 그래프가 알아서 갱신됩니다.

- 보는 곳: https://rtdatalab.github.io/airdeep-wonju-report/
- 저장소: https://github.com/RTdataLab/airdeep-wonju-report.git

## 파일 구성

- `index.html` — 화면. 제목과 분석 코멘트, 표 문구는 여기서 고칩니다.
- `app.js` — 그래프 코드. 데이터 파일명과 공휴일이 맨 위에 모여 있습니다.
- `style.css` — 디자인. 보통 손댈 일은 없습니다.
- `data/` — 데이터 CSV. 매달 이 폴더만 교체합니다.

data 폴더는 그래프 하나에 CSV 하나로 대응합니다.

| 파일 | 그래프 |
|---|---|
| 3-1.csv | 구역별 일평균 실내온도 |
| 3-2.csv | 온습도계 설치 구역 온도 |
| 4-1.csv | 구역별 누적 가동시간 |
| 5-1.csv | 종합운동장 공간별 가동시간 |
| 5-2.csv | 치악체육관 공간별 가동시간 |
| 5-3.csv | 국민체육센터 층별 가동시간 |
| 5-4.csv | 종합체육관 공간별 가동시간 |
| 6-1.csv | 구역별 종합 분석표 |
| 6-2.csv | 월간 누적 가동시간 TOP5 |

매달 받는 CSV를 위 이름 그대로 덮어쓰면 됩니다.

## 매달 하는 일

실제 작업은 `PROMPT.md`에 정리해 뒀습니다. CSV를 폴더에 두고 그 파일의 프롬프트를
Codex에 붙여넣으면 데이터 교체부터 코멘트 수정, push까지 한 번에 진행됩니다.

손으로 직접 한다면 순서는 이렇습니다.

1. data 폴더의 CSV를 새 파일로 교체
2. index.html의 코멘트 수정 (섹션마다 주석이 달려 있어 위치 찾기 쉽습니다)
3. 공휴일이 있는 달이면 app.js 맨 위 PUBLIC_HOLIDAYS에 날짜 추가 (주말은 자동)

## 처음 한 번만 준비

새 PC라면 Git이 필요합니다. 아래는 직접 명령어를 칠 필요 없이 Codex에게 부탁하면 됩니다.

1. Git 확인 — "Git 깔려있는지 확인하고 없으면 설치 방법 알려줘"
2. 저장소 받기 — "이 저장소 받아줘: https://github.com/RTdataLab/airdeep-wonju-report.git"
3. GitHub 로그인 — 처음 올릴 때 로그인 창이 뜨면 RTdataLab 팀 계정으로 로그인합니다.

## 올리고 확인하기

수정이 끝나면 Codex에게 "커밋하고 push 해줘"라고 하면 됩니다.
1~2분 뒤 라이브 주소를 Ctrl+Shift+R(강력 새로고침)로 열어 확인하세요.
바로 안 바뀌면 대부분 브라우저 캐시 때문이라, 잠시 뒤 다시 새로고침하면 됩니다.

## PDF로 뽑기

상단의 "PDF 한 장으로 저장"은 화면 그대로 세로로 긴 한 장,
"A4로 인쇄"는 A4 여러 장으로 나옵니다. 크롬 인쇄창에서 배경 그래픽을 켜야 색이 함께 나옵니다.

## 참고

- 팀 저장소로 막 옮긴 뒤라면 Settings > Pages에서 main 브랜치로 Pages를 한 번 켜야 라이브가 동작합니다.
- 팀원이 push하려면 RTdataLab 멤버(쓰기 권한)로 등록돼 있어야 합니다.
- 차트가 빈 화면이면 data 폴더의 CSV 파일명이 규칙과 다른지 먼저 확인하세요.
