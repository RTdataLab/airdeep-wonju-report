# 원주시설관리공단 월간 모니터링 리포트 (분리 버전)

단일 HTML을 **HTML / CSS / JS / CSV**로 분리한 버전입니다.
CSV만 고치면 리포트 내용이 바뀝니다.

## 폴더 구성

```
wonju/
├─ index.html      ← 화면 구조 (마크업)
├─ style.css       ← 디자인 (색상·여백·레이아웃·표 너비)
├─ app.js          ← CSV를 읽어 차트를 그리는 코드
└─ data/
   ├─ temp.csv     ← 일자별 구역 온도
   ├─ oper.csv     ← 일자별 "시설·구역" 가동시간
   └─ summary.csv  ← 구역별 종합 분석 (KPI·표·TOP5·누적막대)
```

## 실행 방법

브라우저로 index.html을 더블클릭하면 **동작하지 않습니다.**
(브라우저 보안상 file:// 에서는 CSV를 못 읽습니다.)

폴더 안에서 아래 명령으로 로컬 서버를 켜세요.

```
python -m http.server 8000
```

그다음 브라우저에서 접속:  http://localhost:8000

CSV를 수정하고 저장한 뒤 **새로고침(F5)** 하면 바로 반영됩니다.

## CSV 수정 가이드

### temp.csv — 일자별 온도
- 첫 열은 날짜(1~30), 나머지 열은 구역 이름.
- 한 행 = 하루. 값은 그날의 일평균 온도(℃).
- 측정값이 없는 칸은 비워두면 차트에서 그 지점이 끊깁니다.

### oper.csv — 일자별 가동시간
- 헤더가 "시설·구역" 형식입니다. 예: DB숙소·312호
- 가운뎃점(·)을 기준으로 자동으로 시설별 차트에 묶입니다.
- 시설 이름은 종합운동장 / DB숙소 / 치악체육관 / 국민체육센터 / 종합체육관
  이 5개를 그대로 써야 해당 시설 차트에 들어갑니다.

### summary.csv — 종합 분석 (가장 중요)
- 한 행 = 한 구역. 이 표가 종합 테이블 + TOP5 + 가동일수 차트 + 누적 가동시간 막대를 모두 만듭니다.
- 열: 구역, 총가동, 가동일, 가동률, 평균온도, 제어기, 차트제어기, 일평균, 26도이하, 판정, 조치
- **판정**은 반드시 risk / warn / ok 셋 중 하나
  - risk → 빨강 "우선 점검"
  - warn → 주황 "보정 필요"
  - ok   → 초록 "유지"
- **차트제어기**: 누적 가동시간 막대 위 라벨에 표시할 제어기 수.
  - 비워두면 "제어기" 열의 값을 그대로 사용합니다.
  - 실제 설치 대수와 차트에 보여줄 수가 다를 때만 채우세요.
    (현재 종합체육관=13, DB숙소=49만 채워져 있고 나머지는 비어 있음)
- **조치** 문장에 쉼표가 들어가면 그 칸 전체를 큰따옴표로 감싸세요.
  예: "312호, 306호 중심 점검"

## 주의

- CSV는 반드시 **UTF-8**로 저장하세요. (엑셀: 다른 이름으로 저장 → CSV UTF-8)
  안 그러면 한글이 깨집니다.
- 상단 "주의가 필요한 항목" 문구와 차트 밑 설명글, 시설별 기기 현황 카드(상단 숫자)는
  표가 아니라 글/고정 수치라 CSV가 아니라 index.html에서 직접 수정합니다.
- 온도·가동시간(temp/oper)의 일자별 값은 원본 시안의 곡선을 그대로 옮긴 것입니다.
  실제 측정값이 준비되면 같은 형식으로 덮어쓰면 됩니다.
  summary.csv의 종합 수치(총가동·가동률·평균온도 등)는 원본 실측값입니다.
