# 📊 원주시설관리공단 — 월간 모니터링 리포트

실내 온습도·냉난방기 가동 분석 리포트입니다.
**HTML + CSS + CSV**로 만든 정적 웹페이지라, 데이터(CSV)만 바꾸면 차트가 자동으로 갱신됩니다.
어렵게 느껴지면 **Codex에게 그대로 말로 시키면** 대부분 알아서 해줍니다.

🔗 **라이브 주소(보는 곳):** https://rtdatalab.github.io/airdeep-wonju-report/
📦 **저장소 주소:** https://github.com/RTdataLab/airdeep-wonju-report.git


> ⭐ **매달 작업은 [`PROMPT.md`](PROMPT.md) 의 복붙용 프롬프트를 쓰세요.** CSV만 주고 그 글을 Codex에 붙여넣으면 끝납니다.

---

## 🗂 파일 구조 (뭘 어디서 고치나)

```
index.html   ← 리포트 화면. 제목·코멘트(분석 글)·표 문구를 여기서 수정
app.js       ← 차트 코드. 데이터 파일명·공휴일을 맨 위에서 관리
style.css    ← 디자인 (보통 건드릴 일 없음)
favicon.png  ← 브라우저 탭 아이콘
data/        ← 데이터 CSV. ★ 매달 여기만 교체하면 됨 ★
```

### data 폴더 — 그래프 1개 = CSV 1개

| 파일명 | 그래프 |
|---|---|
| `3-1.csv` | 구역별 일평균 실내온도 |
| `3-2.csv` | 온습도계 설치 구역 온도 |
| `4-1.csv` | 구역별 누적 가동시간 |
| `5-1.csv` | 종합운동장 공간별 가동시간 |
| `5-2.csv` | 치악체육관 공간별 가동시간 |
| `5-3.csv` | 국민체육센터 층별 가동시간 |
| `5-4.csv` | 종합체육관 공간별 가동시간 |
| `6-1.csv` | 구역별 종합 분석표 |
| `6-2.csv` | 월간 누적 가동시간 TOP5 |

> 📌 **규칙:** 매달 데이터팀이 주는 CSV를 위 파일명 **그대로 덮어쓰기**만 하면 됩니다.
> 파일명이 다르면 Codex에게 *"data 폴더 규칙에 맞게 이름 바꿔서 넣어줘"* 라고 하세요.

## 🔁 매달 하는 작업 (핵심)

### ① 데이터 교체
데이터팀 CSV를 `data/` 폴더에 **같은 이름으로 덮어쓰기**.
→ Codex에게: *"데이터팀이 준 이 CSV들을 data 폴더 규칙에 맞게 교체해줘"*

### ② 분석 글(코멘트) 수정
`index.html`에는 **섹션별 주석**(`<!-- 3. ... -->`)이 달려 있어 위치 찾기 쉬워요.
→ Codex에게: *"이번 달 데이터 기준으로 KPI 숫자랑 코멘트(분석 글) 다시 써줘"*

### ③ 공휴일 (있는 달만)
`app.js` 맨 위 `PUBLIC_HOLIDAYS` 줄에 공휴일 날짜만 추가 (주말은 자동 계산).
→ Codex에게: *"6월 현충일(6/6)을 공휴일에 추가해줘"* 처럼.

## 🖨 PDF로 뽑기

리포트 상단 버튼 2개:
- **📄 PDF 한 장으로 저장** — 화면 그대로 세로로 긴 한 장 (공유·열람용)
- **🖨 A4로 인쇄** — A4 여러 장 (종이 출력용)

→ 크롬 인쇄창에서 **"배경 그래픽" 체크**를 꼭 켜세요 (안 켜면 색·배경이 빠짐).

## ⚙️ 관리자 1회 세팅 — GitHub Pages 켜기

팀 저장소로 막 옮겼다면 라이브 사이트가 꺼져 있을 수 있어요. 한 번만:
GitHub 저장소 → **Settings → Pages → Branch: `main` 선택 → Save**
→ 1~2분 뒤 `https://rtdatalab.github.io/airdeep-wonju-report/` 가 열립니다.
